**4. API Reference (DOC-02):**
Document all 3 public functions with their exact signatures from __init__.py:

- `xlsx_to_ream(path: str | Path, options: ReamOptions | None = None) -> str` -- converts file path to REAM. Args: path (str or Path to XLSX file), options (optional ReamOptions). Returns: REAM-formatted string. Raises: InvalidWorkbookError, ConversionError.

- `bytes_to_ream(data: bytes, options: ReamOptions | None = None) -> str` -- converts raw XLSX bytes. Args: data (raw bytes of XLSX workbook), options (optional ReamOptions). Returns: REAM-formatted string. Raises: InvalidWorkbookError, ConversionError.

- `file_to_ream(stream: IO[bytes], options: ReamOptions | None = None) -> str` -- converts file-like object. Args: stream (binary file-like object), options (optional ReamOptions). Returns: REAM-formatted string. Raises: InvalidWorkbookError, ConversionError.

Document ReamOptions as a frozen dataclass with fields:
| Field | Type | Default | Description |
|-------|------|---------|-------------|
| max_rows_per_sheet | int | 500 | Maximum rows to convert per sheet |
| force_col_selectors | bool | False | Force A1-style column selectors on every cell |
| collapse_rows | bool | False | Merge identical adjacent rows into row-span records |

**5. CLI Usage (DOC-03):**
Document the `ream-xlsx` command and `python -m ream_xlsx` alternative.

Basic usage: `ream-xlsx input.xlsx`
Output to file: `ream-xlsx input.xlsx -o output.txt`

Flags table:
| Flag | Default | Description |
|------|---------|-------------|
| `-o`, `--output FILE` | stdout | Write output to FILE instead of stdout |
| `--max-rows N` | 500 | Maximum rows per sheet |
| `--collapse-rows` | off | Collapse identical adjacent rows |
| `--force-col-selectors` | off | Force column selectors in output |
| `--version` | - | Show version and exit |

Show example invocations:
```bash
# Convert with all options
ream-xlsx workbook.xlsx --max-rows 100 --collapse-rows --force-col-selectors -o output.txt

# Via python module
python -m ream_xlsx workbook.xlsx
```

**6. Error Handling (DOC-05):**
Document the exception hierarchy:
- `ReamError` -- base exception for all package errors
- `InvalidWorkbookError(ReamError)` -- raised when input is not a valid XLSX file (missing file, corrupted data, wrong format)
- `ConversionError(ReamError)` -- raised when conversion logic fails internally

Show error handling example:
```python
from ream_xlsx import xlsx_to_ream, ReamError, InvalidWorkbookError

try:
    text = xlsx_to_ream("workbook.xlsx")
except InvalidWorkbookError as e:
    print(f"Bad input: {e}")
except ReamError as e:
    print(f"Conversion error: {e}")
```

Note: CLI exits with code 1 and prints error to stderr on any ReamError.

**7. Developer Guide (DOC-04):**
Package layout:
```
ream_xlsx/
  __init__.py       Public API (xlsx_to_ream, bytes_to_ream, file_to_ream)
  _options.py       ReamOptions dataclass
  _exceptions.py    Exception hierarchy
  _io.py            I/O adapters (load workbooks from path/bytes/stream)
  _converter.py     Core conversion logic
  _cli.py           CLI entry point (Click command)
  __main__.py       python -m ream_xlsx support
  py.typed          PEP 561 type marker
```

Convention: modules prefixed with `_` are internal. Public API is only what `__init__.py` exports (listed in `__all__`).

Running tests:
